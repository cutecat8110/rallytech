# 設計規則整理

這份文件保留「怎麼使用」與「設計意圖」兩部分，方便後續切版與設計協作。

這份內容在整理前有幾個特徵：

- 同一組內容有重複匯出
- 色票數值與說明文字混在一起
- 部分內容順序錯位
- `Shadows` 區塊原始匯出順序有錯位，已依補充數值整理

後續使用時，建議分工如下：

- `docs/design/*.json`：當作數值與 token 來源
- 本文件：當作使用原則與設計規則來源

## 1. 色彩使用原則

### 品牌色

| 角色        | 用途                         | 使用原則                                                  |
| ----------- | ---------------------------- | --------------------------------------------------------- |
| `Primary`   | 主操作、提交、購買、核心訊息 | 優先用在主要行動按鈕 與最重要的互動，不要同頁大量分散使用 |
| `Secondary` | 側欄、頁尾、深色容器         | 用來穩住版面邊界與整體重心，適合做深色底容器              |
| `Accent`    | 點綴、徽章、提醒             | 低頻使用，保留給視覺亮點與局部提醒，避免搶走品牌主色焦點  |

### 中性色

| 角色            | 用途                       | 使用原則                                               |
| --------------- | -------------------------- | ------------------------------------------------------ |
| `Surface`       | 文字、背景、邊框、次要操作 | 優先拿來建立閱讀層次與介面骨架，再搭配品牌色做重點強化 |
| `Black / White` | 最高對比情境               | 保留給不可妥協的判讀場景，不建議日常濫用純黑純白對撞   |

### 狀態色

| 角色      | 用途                         | 使用原則                                 |
| --------- | ---------------------------- | ---------------------------------------- |
| `Success` | 完成、成功、正向回饋         | 只負責成功狀態，不混作品牌主色           |
| `Info`    | 提示、說明、一般通知         | 低急迫資訊，避免做成高壓警示感           |
| `Warning` | 注意提醒、待處理事項         | 屬非阻斷提醒，應和 `Danger` 保持明顯區隔 |
| `Danger`  | 刪除、錯誤、駁回、破壞性操作 | 用於高風險或負向結果，需第一眼提高警覺   |

### 色彩使用順序

1. 先選語意，再選色階。
2. 主要按鈕與核心操作優先用 `Primary`。
3. 版面骨架、文字、邊框優先用 `Surface`。
4. `Accent` 只做少量點綴，不拿來承接大面積功能色。
5. 狀態色只表達狀態，不兼任品牌色。

### 科技感定調（2026-03 定版）

- 定版版本：`RALLY-COLOR-LOCK-v1`（見 `04-色彩定版與治理.md`）。
- 主色調改為「科技青綠 + 石墨深色骨架 + 低亮藍灰層次 + 低頻金屬暖點」。
- `Primary`：偏科技感的青綠階，負責主要互動與核心行動按鈕。
- `Secondary`：石墨深色階，負責深色區塊、頁尾與大型容器重心。
- `Secondary` 收斂規則：頁面骨架優先使用 `700-950`，其中 `900/950` 作為深色背景主力。
- `Secondary` 的 `50-400` 不作為常規頁面底色，僅保留在低頻輔助情境。
- `Info`：作為資訊提示與低亮藍灰層次使用，不作高亮大面積主視覺色。
- `Accent`：銅色階，只保留在低頻亮點（建議單頁視覺占比 < 10%）。
- 建議配色節奏：`Secondary/Neutral 70%`、`Primary 20%`、`Info + Accent 10%`。

### 定版補充（執行層）

- 主要行動按鈕（白字）預設採用：`bg-primary-700 hover:bg-primary-600 active:bg-primary-800`。
- 若需改色，必須同步更新 `main.css` 與 `docs/design/tokens` 鏡像檔。

### Secondary 與 Neutral 的關係

- `Secondary` 不與 `Neutral` 合併，也不建議做成同色系。
- `Neutral` 負責文字、邊框、一般表面骨架；`Secondary` 負責深色節奏與品牌科技感重心。
- 若 `Secondary` 與 `Neutral` 同化，深色段會退化為一般灰黑，品牌識別與張力會下降。

## 2. 字體系統

### 字體家族

| 類型 | 字體           | 說明                             |
| ---- | -------------- | -------------------------------- |
| 中文 | `Noto Sans TC` | 承接主要中文閱讀內容             |
| 英文 | `Inter`        | 承接英文標題、介面文字與英文內文 |
| 數字 | `Inter`        | 維持數字資訊一致性與乾淨節奏     |

設計意圖：

- 先確保跨語系一致，再延伸品牌風格
- 中文、英文、數字分工清楚，但實作時應優先統整成同一組雙語 fallback stack

### 字重

| Token     | 數值 | 用途                            |
| --------- | ---- | ------------------------------- |
| `Regular` | 400  | 內文、一般敘述                  |
| `Medium`  | 500  | 次標題、Label、需稍微強調的資訊 |
| `Bold`    | 700  | 主標題、強調訊息                |

原則：

- 只保留 400 / 500 / 700 三層
- 重點是資訊權重清楚，不追求過多細分

### 字距

| Token     | 值        | 用途                         |
| --------- | --------- | ---------------------------- |
| `Tighter` | `-0.02em` | 大標題、需要更凝聚的視覺張力 |
| `None`    | `0`       | 一般文字預設                 |
| `Wider`   | `0.02em`  | 短字、Label、按鈕文字        |

### 行高

| Token     | 值     | 用途                 |
| --------- | ------ | -------------------- |
| `compact` | `120%` | 短字、Display、Label |
| `tight`   | `130%` | Headline、Title      |
| `normal`  | `160%` | 內文                 |
| `relaxed` | `180%` | 長文與較長段落       |

原則：

- 短字偏緊，讓主標更集中
- 長文偏鬆，讓閱讀更穩定

| `Display`  | `L`   | 48   | 700    | `-0.02em`      | `120%`      | 大型主視覺標題     |
| `Display`  | `M`   | 40   | 700    | `-0.02em`      | `120%`      | 主視覺次階標題     |

| `Headline` | `S`   | 20   | 700    | `0`            | `130%`      | 區塊短標           |
| `Title`    | `L`   | 18   | 500    | `0`            | `130%`      | 次層標題           |
| `Title`    | `M`   | 16   | 500    | `0`            | `130%`      | 卡片標題、列表標題 |
| `Body`     | `M`   | 16   | 400    | `0`            | `160%`      | 一般內文           |

使用順序：

1. 頁面英雄區先用 `Display`
2. 頁面與區塊主標用 `Headline`
3. 卡片、模組、次層導引用 `Title`
4. 一般敘述用 `Body`
5. 元件內短字用 `Label`

## 4. Foundations

### Icons 圖示

| 規則      | 共識                                        |
| --------- | ------------------------------------------- |
| Icon 集合 | `@iconify-json/ic`                          |
| 風格      | 僅使用 `baseline`                           |
| 命名格式  | `i-ic-baseline-*`                           |
| 禁止混用  | 不使用 `outline` / `round` / 其他 icon 集合 |

原則：

- 圖示先以功能語意為主（Navigation、Action、Search、Security、Alert）。
- 同一頁避免重複堆疊過多裝飾 icon，優先保留資訊與操作關鍵點。
- 新增 icon 前先確認現有可重用項；若未使用，應同步清理。

### Radius 圓角

| Token  | 用途                     |
| ------ | ------------------------ |
| `XS`   | 細框與低存在感元素       |
| `S`    | 按鈕、輸入框等控制元件   |
| `M`    | 卡片、面板等標準容器     |
| `L`    | Modal、Drawer 等浮層容器 |
| `XL`   | 主視覺與高層級容器       |
| `Full` | Pill、Badge、Avatar      |

原則：

- 圓角代表層級與觸感，不是裝飾
- 不同元件應固定使用既定半徑，不自由發明新數值

### Shadows 陰影

| Token      | 用途                                                                       | 陰影值                                                                                       |
| ---------- | -------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| `Shadow-1` | 貼地內容、輸入框預設態、分隔區塊                                           | `0px 1px 2px -1px #09090B1F`, `0px 2px 4px 0px #09090B14`, `0px 4px 10px 2px #09090B0D`      |
| `Shadow-2` | 低層浮起（卡片 hover、互動卡片）、中層跟隨（下拉面板、氣泡層、滾動後頁首） | `0px 3px 8px -3px #09090B1F`, `0px 6px 14px 0px #09090B14`, `0px 10px 24px 6px #09090B0D`    |
| `Shadow-3` | 高層覆蓋（對話框、抽屜、全域浮層）                                         | `0px 6px 16px -6px #09090B1F`, `0px 10px 24px 0px #09090B14`, `0px 16px 40px 10px #09090B0D` |

原則：

- 陰影只表達層級與聚焦，不負責裝飾
- 單一畫面建議不超過兩個陰影層級
- 系統以向下投影為主
- `Shadow-1` 給貼地與低存在感元素，`Shadow-2` 給一般浮起元件，`Shadow-3` 只留給高層覆蓋
- 程式實作統一使用 `shadow-sys-sm/md/lg`（或對應語意 class），正式頁面避免 `shadow-[...]` 任意值
- 專案自訂 helper class 統一採 `*-sys-*` 命名，方便辨識設計系統語意 class 與原生 utility 的邊界

### Space 間距

| Token       | 用途                                    |
| ----------- | --------------------------------------- |
| `Space-4`   | Icon + 文字、徽章、迷你元件最小貼合距離 |
| `Space-8`   | 基礎元件內距與短元素分隔                |
| `Space-12`  | 高密度後台場景                          |
| `Space-16`  | 全站標準距離、卡片內距、手機最小邊距    |
| `Space-24`  | 區塊、卡片、網格間的標準群組距離        |
| `Space-40`  | 大區塊分段與轉場留白                    |
| `Space-80`  | Desktop 容器左右邊距、大型區塊外距      |
| `Space-128` | 首屏主視覺與大型活動版位的主留白        |

原則：

- 間距負責資訊節奏，不只是留白
- 頁面密度要靠既定尺度維持，不靠感覺微調

### Breakpoints 斷點

| 裝置寬度 | 裝置類型 | 左右留白 |
| -------- | -------- | -------- |
| `1920`   | 桌機     | `80`     |
| `1280`   | 筆電     | `80`     |
| `1024`   | 橫向平板 | `40`     |
| `768`    | 直向平板 | `40`     |
| `640`    | 手機     | `16`     |
| `375`    | 手機     | `16`     |

原則：

- 採滿版容器（Fluid）
- 主要靠左右留白調整閱讀節奏，不急著縮成固定窄版心

## 5. Layout 佈局

### Grid 網格

| 條件      | 欄數  |
| --------- | ----- |
| `>= 1024` | 12 欄 |
| `>= 768`  | 8 欄  |
| `< 768`   | 4 欄  |

原則：

- 模組沿欄線排列
- 跨裝置切換時，優先維持節奏一致，而不是逐頁重排

### Header Height 頁首高度

| Token     | 高度 | 用途                         |
| --------- | ---- | ---------------------------- |
| `XL`      | 128  | 一頁式與活動頁首屏的大型頁首 |
| `L`       | 80   | 多入口頁首                   |
| `M`       | 64   | 前台標準頁首與 Sticky Header |
| `Sub Nav` | 48   | 前台子選單與次層導覽         |
| `Admin`   | 56   | 後台頁首與工具列             |

## 6. 轉成前端規則的方式

這份文本後續建議這樣對應：

- 色彩說明對應到 semantic tokens
  `Primary / Surface / Success / Danger` 這些角色，不直接在元件裡寫死 hex
- 字體說明對應到 typography utilities
  `Display / Headline / Title / Body / Label`
- 基礎規則對應到 layout tokens
  `radius / space / breakpoints / grid / header height`

## 7. 切版時的固定原則

1. 顏色先選語意角色，再選色階。
2. 文字先選樣式角色，再決定內容長短。
3. 區塊間距優先使用既定 `Space` 尺度。
4. 卡片、按鈕、表單的圓角固定沿用系統半徑。
5. `Accent` 與狀態色都低頻使用，避免全頁過度花。
