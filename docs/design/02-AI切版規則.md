# AI 切版規則

這份文件是給 AI / 前端實作時的落地規則。

設計意圖看 [01-設計規則整理.md](./01-設計規則整理.md)，這份專注在 Nuxt UI 的實作分層。

## 核心原則

以 Nuxt UI 三層覆寫為主，不使用 Base 包裝元件當預設入口：

1. 全域預設：`app.config.ts`
2. 頁面或區塊覆寫：`<UTheme :ui="...">`
3. 單點例外：元件上的 `:ui` / `class`

## Nuxt UI First（核心宗旨）

1. 能用 Nuxt UI 就不用手刻：互動元件預設必須採 Nuxt UI。
2. 客製順序固定：`app.config.ts` → `<UTheme>` → 元件 `:ui/class`。
3. 只有在 Nuxt UI 無對應能力時，才允許自製 UI。
4. 若採自製 UI，需在元件以中文註釋說明「為何 Nuxt UI 不適用」。
5. 不得為了短期方便，改以原生 `button/input/select/dialog` 重做 Nuxt UI 既有能力。

## 主要使用順序

1. 先看 [01-設計規則整理.md](./01-設計規則整理.md) 確認語意規則。
2. 先判斷需求是全域、頁面區域、或單一元件。
3. 依分層調整，不跨層硬寫。
4. 頁面檔優先使用 Tailwind token class（例如 `bg-primary-700`、`text-neutral-700`）。

## 實作來源

- `app/assets/css/main.css`
  全域 token、typography utilities、layout helpers、shadow tokens。
- `app.config.ts`
  Nuxt UI 全域預設樣式（button / card / input / formField 等）。
- `app/pages/**`
  用 `<UTheme>` 做頁面或區塊覆寫；單點再用 `:ui` / `class`。

## 分層指南

- 全域 (`app.config.ts`)
  放 80% 共通規則，例如：圓角、字重、常用大小、預設 variants。
- 頁面 (`UTheme`)
  放某一頁或某個區塊的統一差異，例如：登入頁按鈕全改圓角、特定區塊卡片風格。
- 單點 (`:ui` / `class`)
  放單一元件例外，例如：某顆按鈕要特殊 icon slot 或特殊 hover。

## 顏色規則

- 色彩定版版本：`RALLY-COLOR-LOCK-v1`（見 `docs/design/04-色彩定版與治理.md`）
- `Primary` 用在主要行動按鈕 與核心互動
- `Secondary` 用在深色容器與版面分層
- `Secondary` 頁面骨架優先使用 `700-950`（尤其 `900/950`）
- `Secondary` 的 `50-400` 僅低頻使用，不作常規頁面底色
- `Accent` 低頻使用，不作為主體功能色
- `Success / Warning / Danger` 僅作狀態語意
- `Info` 僅作資訊提示與低亮藍灰層次，不作高亮大面積主色
- `Secondary` 與 `Neutral` 不合併：`Neutral` 管閱讀骨架，`Secondary` 管深色品牌重心
- 主要行動按鈕（白字）預設 `bg-primary-700 hover:bg-primary-600 active:bg-primary-800`
- `UButton` 的 `color="primary" + variant="solid"` 需維持白字（必要時僅可顯式加 `text-sys-inverse`）
- 頁面層優先使用 token class，不在 template 寫 `var(...)`

## 文字規則

- Hero：`Display`
- 頁面與區塊主標：`Headline`
- 次標與卡片標題：`Title`
- 一般段落：`Body`
- 按鈕與欄位標籤：`Label`

## 圖示規則

- 統一使用 `@iconify-json/ic` 的 `baseline` 系列
- 命名只用 `i-ic-baseline-*`
- 不混用 `outline`、`round` 或其他 icon 集合
- 新增 icon 請先確認現有可重用；不再使用的 icon 要同步移除

## 禁止事項

- 不在頁面檔直接寫 hex color
- 不在頁面檔直接寫 `var(--...)` 作為 class
- 不自行新增 token 色階或改動現有色票語意
- 不在 `app.config.ts` 加 route 條件判斷樣式
- 不把頁面特殊需求硬塞回全域設定
- 不讓單點 `:ui` 大量散落而沒有收斂策略
- 不在 Nuxt UI 可覆蓋的情況下，額外手刻同功能互動元件
- 不使用高優先級全域連結字色 reset（例如 `a { color: inherit; }`）
