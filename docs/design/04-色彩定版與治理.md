# 色彩定版與治理（Color Lock）

本文件定義 RallyTech 網站色彩的「定版版本」與後續變更流程。

## 1. 定版資訊

- 版本：`RALLY-COLOR-LOCK-v1`
- 定版日期：`2026-03-19`
- 適用範圍：
  - `/` 首頁
  - `/design-system`
  - `/design-system-color-lab`

## 2. 單一來源（Source of Truth）

色彩數值以以下順序為準：

1. `app/assets/css/main.css`（執行期唯一真實值）
2. `docs/design/tokens/color-semantic.json`（語意 token 鏡像）
3. `docs/design/tokens/color-primitives.json`（Figma 原始色階鏡像）

若三者不一致，必須先修正到一致後才能合併。

語意鏡像命名補充（沿用 Figma 匯出）：

- `color-semantic.json` 的 `Accent` 對應程式端 `accent`
- `color-semantic.json` 的 `Surface` 對應程式端 `neutral`

## 3. 定版色彩方向

- `Primary`：科技青綠，負責主操作與品牌辨識。
- `Secondary`：石墨深色骨架，負責深色區塊重心（非一般中性色）。
- `Info`：低亮藍灰，僅用於深色背景層次與資訊提示。
- `Accent`：低頻金屬暖點，避免大面積鋪色。

## 4. 使用鎖定規則

1. 深色區塊只用 `Secondary 700-950`，主背景優先 `900/950`。
2. `Secondary 50-400` 不作常規頁面底色，只作低頻輔助。
3. 主要行動按鈕（白字）預設 `bg-primary-700 hover:bg-primary-600 active:bg-primary-800`。
4. `Info` 不可作高亮大面積主視覺色塊，只能低頻層次使用。
5. 禁止在頁面直接新增散落 hex；必須走 token。

## 5. 變更流程（未來改色必走）

任何色彩變更都必須同步更新以下檔案：

1. `app/assets/css/main.css`
2. `docs/design/tokens/color-semantic.json`
3. `docs/design/tokens/color-primitives.json`
4. `app/components/page/design-system/ColorSection.vue`
5. `docs/design/01-設計規則整理.md`（或本文件）

並完成以下檢查：

1. `npm run lint`
2. `npm run typecheck`
3. 手動檢視 `/`、`/design-system`、`/design-system-color-lab`

## 6. 變更門檻

- 未經明確需求與評估，不進行「風格性」色彩改動。
- 若要改色，需先說明：
  - 目標問題（可讀性、品牌一致性、轉換率、可及性）
  - 影響範圍
  - 回退策略
