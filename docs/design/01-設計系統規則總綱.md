# 設計系統規則總綱

最後更新：2026-03-31

## 文件目的

- 統一定義設計系統語意規則與治理方式。
- 提供設計與前端協作時的單一來源，避免品牌表現與實作脫節。

## 適用角色

- 設計：確認品牌色彩、字體與 foundations 的固定邊界。
- 前端：確認 token 使用規則與執行期真實值來源。
- AI agent：在切版與視覺調整時，先遵守這份主檔再做例外處理。

## 1. 單一來源與治理

色彩與 token 來源順序：

1. `app/assets/css/main.css`（執行期唯一真實值）
2. `docs/design/tokens/color-semantic.json`
3. `docs/design/tokens/color-primitives.json`

若三者不一致，先修正一致再合併。

## 2. 設計方向

- 視覺不直接照抄參考站，而是承接其資訊層級與畫面節奏後，回到雷力科技的品牌語氣。
- 版面可承接參考站骨架，但色彩、字體、按鈕與互動重心要以自家設計系統為準。
- 整體方向以「工業 B2B、專業可信、可交付」為主，不走過度裝飾或模板感路線。

## 3. 色彩系統（RALLY-COLOR-LOCK-v1）

- `Primary`：科技青綠，主操作與品牌辨識
- `Secondary`：石墨深色骨架，用於深色容器
- `Info`：低亮藍灰，用於資訊提示與低頻層次
- `Accent`：低頻點綴，不承接主功能色

鎖定規則：

1. 深色區塊使用 `Secondary 700-950`，主背景優先 `900 / 950`
2. `Secondary 50-400` 不作常規頁面底色
3. 主要按鈕（白字）固定 `bg-primary-700 hover:bg-primary-600 active:bg-primary-800`
4. 禁止在頁面散落新增 hex，必須走 token

## 4. 字體與文字層級

字體策略：

- 中文：`Noto Sans TC`
- 英文與數字：`Inter`
- 字重：`400 / 500 / 700`

文字樣式：

- `Display`：首屏與最高層級主標
- `Headline`：頁面與區塊主標
- `Title`：次層標題與卡片標題
- `Body`：一般內文
- `Label`：按鈕與欄位短字

## 5. Foundations 與 Layout

- Icons：`@iconify-json/ic` baseline（`i-ic-baseline-*`）
- Radius：沿用系統半徑，不自建任意數值
- Shadow：僅表達層級，單頁不超過兩層陰影語意
- Space：沿用固定尺度（4 / 8 / 12 / 16 / 24 / 40 / 80 / 128）
- Breakpoints：1920 / 1280 / 1024 / 768 / 640 / 375
- Grid：Desktop 12、Tablet 8、Mobile 4

## 6. 變更流程（設計規則）

任何色彩或 token 變更需同步更新：

1. `app/assets/css/main.css`
2. `docs/design/tokens/color-semantic.json`
3. `docs/design/tokens/color-primitives.json`
4. 相關展示元件（如 design-system 色彩區塊）

並完成：

1. `pnpm lint`
2. `pnpm typecheck`
3. 手動檢視首頁與 design-system 相關頁面
