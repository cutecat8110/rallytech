## 判斷摘要

- Hero 選擇明亮控制室，而不是首頁既有 `ote-control-room-bg`，原因是既有圖有過多人物與螢幕雜訊；頁首需要可承接參考站矮版 hero 的穩定背景。
- 主內容延續參考站的大圖 + 窄圖組合，但改用跨系統整合語境：控制室代表 unified operational view，機櫃代表 Historian / data backbone。
- `Vibration Analysis` 沒有直接採用 generic motor photo；本輪先用儀表 / condition monitoring 類圖像承接「可判讀的狀態訊號」，避免圖像看起來像單一維修服務。
- 曾檢視 `alarm-monitoring/detail-hero-control-room-alarm-screens.jpg`，但因人物衣服有明顯企業字樣，未作為本頁主要畫面。
- 本輪未新增外部下載；API key 缺席時不以截圖或 hotlink 作正式素材流程，先重用已本地化且有來源紀錄的免費圖庫素材。
