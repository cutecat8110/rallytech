# 首頁媒體素材來源

最後更新：2026-04-08

## Owner boundary

這份文件現在是 appendix only。
它保留人類可讀的素材來源、授權與選圖脈絡，但不再作為 `public/images` 的 canonical inventory owner。
真正的 canonical inventory、命名與 lifecycle status 請以 `data/image-assets/catalog.json` 為準。

## 選圖原則

- 不再以泛工業現場照作為首頁主視覺。
- 改以參考站的四種圖片角色反推變體：
  1. `Hero`：高張力控制室 / 監控空間
  2. `About`：高科技建築或現代設施立面
  3. `OTE`：機櫃、伺服器或資料整合場景
  4. `Mission`：單一技術角色 / 現代工程工作情境
- 所有素材皆為可商用開放授權來源，並本地化存放於 `public/images/stock/home/`。

## 備註

- 這一輪以「圖片角色 + 構圖類型 + 科技感」對齊參考站，不直接複製來源站資產。
- 若後續要更貼近雷力的 `SCADA / PLC / PI / Historian / 工業網路` 真實場域，建議優先升級 `Hero`、`OTE`、`Mission cutout` 三張圖。
- Hero 的 AI 候選與 prompt 歷程另記錄於 `data/nano-banana/manifests/home-hero.json` 與 `data/nano-banana/prompts/`；目前正式站仍以 stock Hero 為預設 live source。
- About 的 AI execution path 採兩層：
  - `about-primary`：`data/nano-banana/manifests/about-primary.json`
  - `about-detail`：`data/nano-banana/manifests/about-detail.json`
- Services 的 AI execution path：
  - `services-surface`：`data/nano-banana/manifests/services-surface.json`
- Mission 的 AI execution path 採兩層：
  - `mission-square`：`data/nano-banana/manifests/mission-square.json`
  - `mission-cutout`：`data/nano-banana/manifests/mission-cutout.json`
- 若正式上線需要更嚴格的授權與 attribution 流程，可再補一份法務版素材清單。
