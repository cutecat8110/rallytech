# 首頁視覺真相與殘留判讀

最後更新：2026-04-01  
參考來源：`https://pentagoneng.com/`

## 目的

- 固定首頁視覺真相擷取方式，避免只看文字或破圖框就開始切版。
- 記錄已確認的壞圖、模板殘留與不應照抄的內容。
- 作為首頁高擬真重構時的 delta notes 基線。

## Capture 指令

```bash
npm run capture:reference:home
npm run capture:local:home
```

輸出位置：

- `pentagonClone/captures/reference`
- `pentagonClone/captures/local`

命名規則固定為：

- `home_loaded_1440.png`
- `home_loaded_390.png`
- 對應的 `.images.json` 與 `.summary.json`

## 真相擷取規則

- 進頁後必須先增量滾動到最底，再回到頂部。
- 每次滾動都等待 lazy image 與 scroll-triggered content 載入。
- 只有完整暖機後的 full-page screenshot 才能當作比對基線。
- Desktop 與 mobile 都要保留一組參考截圖與一組本機截圖。

## 已確認判讀

### 應視為壞圖，不是 UI 設計

- About 區塊的 `About Image`
  - 其中數張來自外部 template asset，已壞鏈。
  - 判讀：`missing-content-image`
- Services 區塊的部分 `Icon Image`
  - 參考站 icon slot 失效，不代表設計上應該留空。
  - 判讀：`missing-content-image`
- Mission / Why Us 區塊的 `Why Us`
  - 外部 template image 失效。
  - 判讀：`missing-content-image`

### 應視為模板殘留，不納入首頁第一輪

- `Just Because You Work Hard You’ll Be Successful.`
- `Read Our Latest Updates`
- `Newsletter`
- footer 中偏模板化的 blog / newsletter 結構

判讀：`template-residual`

### 可保留節奏，但不照抄錯誤

- 服務卡第二、三列存在 title / description 對調問題。
- `Vibration Analysise` 為來源 typo。

處理原則：

- 首頁卡片命名與文案以 `docs/project/03-服務架構與命名決策.md` 與 `docs/references/zh-tw/00-首頁.md` 為準。
- 參考站 typo 與內容錯置只保留在 `source` 層，不回寫到正式實作。

## 首頁重構 implications

- Hero：承接參考站深色大面積主視覺節奏，但不引用來源錯誤素材。
- About：必須保留「文字 + 媒體」雙欄敘事，不能把破圖框當成排版本體；右側推測為雙圖組合，其中一張壞圖位應還原成高科技建築細節圖。
- Services / Process：需補回一致的 icon language，不接受壞圖佔位；Services section 應存在低對比背景圖，不是純平面深色底。
- The One-Touch Experience：參考站是背景影片邏輯，第一輪以背景圖 poster frame 還原，不再以前景 framed image 代替。
- Mission / Why Us：應還原成雙態切換模組（`Our Mission` / `Join Us!`），左下方方圖與文案同步切換，右側保留單一工程角色視覺。
- Footer connector：`Engineering + Design Experts` 不是單純矩形條，左側應有一張向外凸出的圖片連進 footer；圖片左下角為直角，內容本身仍收在 container 內。
- Footer：只承接底部 CTA 收斂感，不回做 newsletter 模板殘留。
