# 文件總覽

`docs/` 採三層結構，並以「單一真實來源」維護規則避免重複。

- [project/README.md](./project/README.md)
  專案策略、IA、服務命名決策與交付規範
- [design/README.md](./design/README.md)
  設計系統規則、AI 切版流程與 Preview 維運規範
- [references/README.md](./references/README.md)
  參考網站整理版文本（`en` / `zh-tw` 雙版本）

## 使用順序

1. 先讀 `project`，確認策略邊界與網站資訊架構。
2. 再讀 `design`，確認設計規則與落地方式。
3. 最後用 `references` 補齊參考文本與在地化草稿。

## 維護原則

- 一個事實只在一份主檔維護，其他檔案只做連結引用。
- 全部文件連結採相對路徑，避免本機絕對路徑失效。
- 參考文本可改寫，但需保留可追溯的來源註記與更新日期。
