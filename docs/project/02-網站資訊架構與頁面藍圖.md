# 網站資訊架構與頁面藍圖

最後更新：2026-03-27

## 文件目的

- 定義第一輪可落地的網站 IA 與頁面角色。
- 提供首頁區塊藍圖與「一站式整合」承接範圍。

## 1. IA 規劃原則

- 第一輪採 `先完整承接，再逐步調整`
- 內容骨架先對齊參考站可運作版本
- 不先引入「應用領域」作為第一層 IA

## 2. 第一層 IA

概念架構：

- `首頁`
- `關於我們`
- `服務項目`
- `一站式整合`
- `聯絡我們`

桌機主選單：

- `關於我們`
- `服務項目`
- `一站式整合`
- `聯絡我們`

導覽規則：

- 桌機選單不另外顯示 `首頁`
- Logo 承擔回首頁功能

## 3. 頁面角色定義

- `首頁`：品牌印象、服務概覽、主要行動引導
- `關於我們`：公司定位、能力敘述、信任訊號
- `服務項目`：承接 8 項服務的總覽與入口（服務清單見 `03`）
- `一站式整合`：整合方案導向頁，非案例頁、非應用場域分類
- `聯絡我們`：主要轉換入口，串接後台聯絡資料

## 4. 首頁區塊藍圖（第一輪）

1. Hero（主標、輔助文案、主要 CTA）
2. 公司定位（角色與價值主張）
3. 核心優勢（3 欄重點）
4. 服務總覽（服務入口）
5. 專案流程（4 步驟）
6. 一站式整合（整合能力入口）
7. 品牌行動區塊
8. 聯絡行動區塊

## 5. 一站式整合承接項目（主清單）

- `Data Integration`
- `Call-Out System Integration and Commissioning`
- `Vibration Analysis`
- `Local/Cloud Data Platform Deployment`
- `Manure Management System Integration`

## 6. 不採用方向（第一輪）

- 不以舊站四大服務當主架構
- 不先做最終最佳分組優化
- 不把 `一站式整合` 直接定義為案例頁
