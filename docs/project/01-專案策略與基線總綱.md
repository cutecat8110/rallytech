# 專案策略與基線總綱

最後更新：2026-03-27

## 文件目的

- 定義專案共同前提與策略邊界。
- 提供後續 IA、設計與開發的單一對齊基準。

## 1. 專案定位

- 專案類型：雷力科技企業官網重構
- 核心目的：清楚呈現服務能力、建立信任、提升聯絡轉換
- 執行節奏：先確立資訊架構與內容骨架，再進入設計與實作

## 2. 主要受眾

- 工程、製程、自動化、儀控、廠務、專案管理等專業客戶
- 主要角色：
  - 業主端工程或廠務窗口
  - 自動化 / 儀控 / 電氣主管與工程師
  - 專案管理、採購或系統整合決策者

## 3. 溝通與語言策略

- 文案語氣：專業、清楚、可信賴，避免過度口語化
- 專業術語可保留：`SCADA`、`HMI`、`PLC`、`DCS`、`PI System`、`Historian`、`I&E`
- 命名採雙層：
  - `短名`：導覽、卡片、查找
  - `正式名`：頁面標題、SEO、雙語對照
- 語言策略：繁體中文優先，英文並列維護

## 4. 範圍邊界

首波必做：

- 首頁、關於我們、服務項目、一站式整合、聯絡我們
- 中英文雙語
- 後台聯絡表單基本查看

首波不做但需預留：

- 代理產品介紹、詢價車、部落格、人才招募

## 5. 轉換與信任目標

主要轉換行為：

- 聯絡諮詢
- 專案初步洽談
- 能力資格確認

核心信任訊號：

- 服務深度與交付能力
- 整合方案可行性
- 安全與品質承諾
- 聯絡窗口與回應效率

## 6. 後台方向

- 首波定位：輕量管理後台，不預設完整 CMS
- 最低需求：
  - 聯絡表單列表
  - 單筆查看（姓名、公司、Email、電話、需求、時間）
- 後續擴充：
  - 狀態、搜尋篩選、匯出、權限、共用資料結構
