# 服務總覽 11 項服務卡截圖 QA

最後更新：2026-04-23

## Objective

確認 `/services` 在新增 11 項服務後，服務總覽卡片能在桌機、平板交界與手機寬度下維持清楚的視覺節奏，並驗證服務卡圖片、文字與 CTA 沒有破圖、裁切或水平 overflow。

## Scope

- 頁面：`/services`
- 區塊：`.services-sys-grid`
- 服務卡數量：11
- 語系：目前預設繁中 public copy
- 本輪不重寫 `docs/rewrite/文案.md`、服務 public copy，也不重新設計服務總覽 IA。

檢查條件：

- `/services` 回應 200。
- 唯一服務 detail link 為 11 個。
- `.services-sys-listing` 渲染 11 張卡。
- 11 張服務卡圖片皆有 `naturalWidth > 0`。
- CTA 皆可見。
- 無水平 overflow。
- 長英文服務名與中文短名未發生可視裁切或覆蓋。

## Visual Findings

- Desktop `1440x900`：2 欄 stagger 節奏成立，左右欄圖片尺寸一致；第 11 張 `告警監控` 留在左欄，右欄留白屬於 11 張 odd card 的自然收尾，沒有破版。
- Tablet `768x1024`：仍維持 2 欄，卡片寬度縮小後，長英文 eyebrow 可換行；中文標題與 CTA 沒有被擠壓。
- Mobile `390x844`：改為 1 欄，所有卡片圖片、標題、描述與 CTA 依序排列；沒有水平 overflow。
- 新增三項服務 `IT 建置`、`資安網路建置`、`離岸風場維運` 的 stock 圖在明暗、裁切與卡片高度上能與既有服務維持一致。
- `遠端監控與資料備援` 在手機寬度下標題換成兩行，屬於正常換行，未裁切也未覆蓋描述。

## Fixes In This Round

- 本輪未調整 `ServicesCardGrid.vue`。截圖 QA 未發現需要同輪修正的 spacing、aspect-ratio、object-position、title width 或 CTA 對齊問題。
- 影像 audit 清理另已對齊 catalog、runtime registry、nano manifest 與 audit helper；詳見本輪程式變更與 `npm run audit:image-assets` 結果。

## Remaining Risks

- QA evidence 以本機 Nuxt dev server 與 Chromium headless 截圖為準；若正式部署環境啟用不同 CDN/image optimization，建議上線前再補 production preview 截圖。
- 11 張卡目前採既有 2 欄 stagger 節奏，未做新版型重設；若未來服務數量再增加，建議另開 IA / visual rhythm pass。
