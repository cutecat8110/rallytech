# 品牌素材缺口與補強清單

最後更新：2026-04-14

## 文件目的

- 釐清目前多語系網站中，哪些問題屬於文案層可優化，哪些問題其實是公司素材不足。
- 提供管理、業務、設計、前端與 AI agent 一份共用的品牌素材盤點基準。
- 避免網站文案在沒有證據的情況下，硬撐出過厚的品牌故事、文化敘事或實績主張。

## 適用範圍

- 首頁
- 關於我們
- 服務總覽
- 8 項服務內頁
- 一站式整合
- 聯絡我們
- FAQ

## 目前不應憑空擴寫的內容

- 企業創立故事、創辦人背景與創業動機
- 公司里程碑、年表、辦公室擴張與團隊成長數字
- 企業文化、核心價值、品牌精神的正式對外版本
- 客戶名單、代表案例、專案量體、跨國據點與產業覆蓋數據
- 認證、獎項、專利、合作夥伴與法規合規主張
- 團隊規模、工程背景、證照與安全紀錄

## 缺口清單

### P1：品牌與信任素材

| 缺口類別         | 目前狀態                                 | 主要影響頁面                     | 若未補齊的限制                                     | 建議蒐集欄位                                          |
| ---------------- | ---------------------------------------- | -------------------------------- | -------------------------------------------------- | ----------------------------------------------------- |
| 公司歷史與里程碑 | 只有 `2014 成立` 可驗證                  | 首頁、關於我們、頁尾             | 只能寫成立年份，無法完整說明品牌發展脈絡           | 創立背景、關鍵年份、重大轉折、擴張節點                |
| 品牌精神與價值觀 | 只有「整合交付、交接可維護」的工作方法   | 首頁、關於我們、招募或合作型文案 | 可描述做事方式，但還無法形成完整品牌精神敘事       | 3-5 條正式對外價值主張、內部常用原句、管理層認可版本  |
| 服務差異化       | 已知承接項目清楚，但為何選雷力仍偏方法論 | 首頁、服務總覽、聯絡我們         | 能說「做什麼」，但還不夠清楚「為何是雷力」         | 相較 SI / 設備商 / 統包商的差異、最常被客戶選中的原因 |
| 可信證據         | 缺少公開案例、成果數據與客戶授權素材     | 首頁、關於我們、服務內頁         | claim 只能停在能力與方法，不宜拉高成績效或領先主張 | 可公開案例、成效指標、客戶類型、可引用的證言          |
| 團隊專業背景     | 無可公開的團隊介紹與專業分工             | 關於我們、聯絡我們               | 難以支撐「工程導向」背後的人與組織可信度           | 團隊角色、背景、主要專長、跨域協作模式                |

### P2：服務補強素材

| 缺口類別 | 目前狀態                                       | 主要影響頁面           | 若未補齊的限制                                     | 建議蒐集欄位                                 |
| -------- | ---------------------------------------------- | ---------------------- | -------------------------------------------------- | -------------------------------------------- |
| 代表場景 | 已有服務名稱，但缺少典型導入場景               | 服務總覽、8 項服務內頁 | 服務說得清楚，但較難快速對到客戶自身情境           | 常見應用場景、客戶提問、啟動條件             |
| 交付邊界 | 已能描述承接內容，但 owner boundary 仍可更清楚 | 服務總覽、聯絡我們     | 客戶知道能談哪些題目，但不一定知道哪些屬於雷力主責 | 承接範圍、協作對象、常見不承接項             |
| 證據層級 | 多數服務仍以方法與流程為主                     | 服務內頁、首頁         | 專業可信，但說服力多停在敘述而非證據               | 可公開設備類型、交付模組、驗收重點、實績摘要 |

### P3：品牌延伸素材

| 缺口類別           | 目前狀態                               | 主要影響頁面   | 若未補齊的限制                       | 建議蒐集欄位                                 |
| ------------------ | -------------------------------------- | -------------- | ------------------------------------ | -------------------------------------------- |
| FAQ 真實題庫       | 現況仍偏 placeholder                   | FAQ、聯絡我們  | FAQ 無法承擔信任建立與預先解答功能   | 真實常見問題、商務疑慮、交付流程問答         |
| 品牌語彙表         | 術語已有基礎，但跨頁仍可能出現寫法漂移 | 全站           | 中英術語與短標可能不夠完全一致       | 正式中文名、英文名、可接受縮寫、避免用字     |
| 視覺與文案共用訊號 | 現況偏靠 copy 承擔品牌定位             | 首頁、關於我們 | 視覺與文案整合後的品牌辨識仍可更完整 | 品牌標語、對外簡介、視覺關鍵詞、照片類型偏好 |

## 頁面影響矩陣

| 頁面       | 目前最能表達的內容           | 目前最缺的內容                 | 判定                 |
| ---------- | ---------------------------- | ------------------------------ | -------------------- |
| 首頁       | 專業角色、交付方法、服務範圍 | 品牌精神、差異化證據、歷史厚度 | `半成熟，可再優化`   |
| 關於我們   | 工作方法、能力結構、交付節奏 | 真實品牌故事、團隊與文化       | `素材不足，不宜硬補` |
| 服務總覽   | 服務架構與分類               | 各服務一眼分辨的差異化證據     | `可靠改稿再提升`     |
| 服務內頁   | 承接範圍、交付價值、專業語氣 | 可公開案例與成效證據           | `目前最成熟`         |
| 一站式整合 | 整合能力與模組範圍           | 真實場景與價值證據             | `方向正確，仍缺證據` |
| 聯絡我們   | 洽詢入口與商務導流           | 低摩擦 proof 與常見導入問題    | `可用，但可再補強`   |
| FAQ        | 幾乎沒有品牌與服務貢獻       | 真實問答素材                   | `素材缺口優先`       |

## 文案問題與素材問題的判定規則

| 情況                                                     | 應判定為                                     |
| -------------------------------------------------------- | -------------------------------------------- |
| 句子太長、像提案稿、直譯感太重、CTA 不夠直覺             | `copy issue`                                 |
| 服務名稱清楚，但場景、邊界與價值還能更精準               | `copy issue` 與 `service proof gap` 共同存在 |
| 想寫公司精神、文化、歷史，但 repo 沒有可驗證內容         | `source-material issue`                      |
| 想寫案例、客戶、成果、認證，但沒有公開素材或授權         | `source-material issue`                      |
| 想把服務說得更有說服力，但缺少實際場景、驗收點或代表模組 | `source-material issue` 優先                 |

## 建議蒐集順序

1. 管理層 / 業務訪談
   - 公司成立背景
   - 為什麼做這家公司
   - 最常對外怎麼介紹雷力
   - 客戶通常為什麼找上門
2. 服務證據盤點
   - 每項服務常見場景
   - 典型交付內容
   - 常見痛點與成功條件
   - 可公開的設備、系統或模組名稱
3. 品牌語彙定稿
   - 中文短標
   - 英文對應
   - 必留術語
   - 應避免的說法
4. 可公開 proof 清單
   - 客戶類型
   - 可引用案例
   - 產業別
   - 認證或合規資料

## 在素材補齊前的文案 guardrails

- 只使用已驗證的成立年份、聯絡資訊、服務架構與工作方法。
- 強調「整合交付、現場可接手、從規劃到交接」這條已被 repo 支持的主軸。
- 不寫未經證實的規模、全球據點、代表客戶、案例成果或市場領先主張。
- 不把 reference 公司故事直接移植成雷力自己的歷史敘事。
- 若需要強化品牌感，優先從語氣、層級、術語一致性著手，不先憑空擴寫品牌故事。
