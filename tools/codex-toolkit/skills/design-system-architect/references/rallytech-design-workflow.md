# RallyTech Design Workflow

## Read Order

1. `app/assets/css/main.css`
2. `app/app.config.ts`
3. `docs/design/01-設計系統規則總綱.md`
4. `docs/design/02-AI切版與Preview維運規範.md`
5. `docs/design/03-Vision-to-Code與Component-Driven規範.md`

## Decision Sequence

1. 這次變更屬於哪一層
2. runtime truth 已經有什麼
3. semantic token 要不要新增或只重映射
4. Nuxt UI 是否已有可承接的 component / slot
5. 規則要不要 promotion
6. 驗證要去哪裡抽查

## Common Deliverables

- semantic token proposal
- ownership map
- Nuxt UI mapping
- promotion / keep-local decision
- validation checklist
